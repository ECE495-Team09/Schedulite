# Schedulite Tutorial

## Overview

In this tutorial, we will go through the main Schedulite flow from start to finish.

We will sign in, create a group, join that group from a second account, create an event, receive the event notification, and respond to it. By the end, you should be able to use Schedulite to help a small group plan an event quickly.

This tutorial is written for a first time user. We will keep the steps simple and direct.

## Prerequisites

Before you start, make sure you have these:

- A device where you can run Schedulite. This can be an iPhone, Android phone, or any laptop or desktop.
- Two google accounts
- (Optional: a second device to sign in with this account, otherwise use a private/incognito browser window to sign in with the second account.)
- Internet access.
- Notification permissions enabled on the device that will receive the event invite.
- `Access to the current Schedulite build, website link, or app from the app store or play store.

It is easiest to do this tutorial with two users.
- User 1 will be the organizer.
- User 2 will be the member who receives the event and responds.

## Setup / Installation

1. On the organizer device, install the app or open the web client.
2. On the member device, install the app or open the web client.
3. Open Schedulite on both devices.
4. Sign in with the two separate google accounts, one on each device.
5. If prompted, allow notifications on the member device.
6. Wait until both accounts reach the home screen.

You should now have two signed in users ready to test the main workflow.

## First Workflow

In this workflow, we will do the main job Schedulite is built for.

We will create a group, add a member, make an event, send the event invite, and record a response.

### Step 1. Sign in as the organizer

On the first account/device, sign in as the organizer.

1. Open Schedulite.
2. Tap the sign in button.
3. Complete the sign in flow.
4. Wait until the home page loads.

You should now be on the main page of the app as User 1. Here you can see your groups and events.

### Step 2. Create a new group

Now we will create a group that will hold our event.

1. On the home page, find the option to create a group.
2. Tap it.
3. Enter a simple group name. For example, Weekend Cricket or Senior Design Team.
4. Click on "Create".

The new group should appear in your group list. Click on the group to open it and make sure it shows you as the owner or organizer. This means the group was created successfully.

### Step 3. Get the group join code or invite link

Next, we need to bring in the second user.

1. Open the group page by clicking on the group if it is not already open.
2. Find the 6 letter group code on the left side of the group info bar.
3. Copy/memorize the code to use it in the second account.

You now have a valid way for another user to join the same group.

### Step 4. Join the group as a member

Now we will join the group as a member.

1. Open Schedulite.
2. Sign in as User 2.
3. Press the "Join Group" button on the home page.
4. Enter the group code to join the same group as User 1.
5. Press "Join Group".

After this, the new group should appear for User 2 as well.

Go back to the organizer account and check the group member list. After refreshing, you should now see the second user in the group.

### Step 5. Create an event

Now we will create the event that the group will respond to.

1. Go to the organizer account.
2. Open the group you created.
3. Press "Create Event".
4. Enter a title.
5. Enter a date and start time.
6. Enter a location.
7. Add a short note or description to describe the event.
8. Press "Create Event" to finish and save the event.

After saving, the new event should appear in the group event list.

Open the event and check that the title, time, and location are correct before moving on.

### Step 6. Receive the event notification

Once the event is created, the member account should receive a notification for it.

1. Look at the member device or tab.
2. Wait for the event notification to appear.
3. You can expand the notification so you can see the possible actions.

The notification should show enough event information for the user to quickly understand what the event is and you should also be able to see response options such as In, Out, or Maybe.

### Step 7. Respond to the event

Now we will record the member response.

1. On the member device, choose a response option.
2. For this tutorial, select In.
3. Submit the response.

After a moment, the response should be saved and you should be able to see your response saved inside the app.

### Step 8. Verify the attendance result

Now we will confirm that the organizer can see the response.

1. Go to the organizer account.
2. Open the event details page.
3. Check the Attendees section.
4. Confirm that the response from User 2 now appears with the response In.

Congratulations, you have completed the main Schedulite workflow.

## Expected Results

If everything worked, you should have the following results:

- Two users signed in to Schedulite.
- One group created by the organizer.
- The second user added to that group.
- One event created with a title, time, location, and optional description.
- A notification sent to the member account.
- A recorded RSVP response from the member.
- The organizer is able to see the updated attendance in the app.

Basically, you should now have a good idea of how Schedulite is supposed to help a group organize an event and collect attendance quickly.

## Troubleshooting

### I cannot sign in

Make sure you are using a valid google account.
If a login window closes too early or does not load, fully close the app and reopen it, then try again.

### The second user cannot join the group

Double check the group code or invite link, a small typo is enough to make the join fail.
If the code still does not work, go back to the organizer account and make another group.

### The event does not appear after I create it

Refresh the page or reopen the group.
If it still does not appear, check that the required fields were filled in, especially the event title and start time.

### The notification does not appear on the member device

Check that notifications are enabled for Schedulite on that device.
Also make sure the device has internet access.

### The RSVP does not update for the organizer

Wait a few seconds and refresh the page.
If the response still does not show, then confirm that the member actually responded.
