# Manual Test Report

**Date (YYYY-MM-DD):** 2026-04-26  
**Tester:** Aryan Banerjee  

**Test Name:** Event CRUD routes — Create, Read, Update, Delete, and Validation in Postman  

**Test Steps (summary):**
1. Started the local backend on `localhost:5000` and connected it to the database.
2. Ensured a test user existed, obtained a valid JWT, and used a valid group ID where the user was a member/admin.
3. In Postman, set `Authorization: Bearer <JWT>` for the authenticated requests.
4. Ran these requests in order:
   - POST Create Event
   - GET Events
   - PUT Update Event
   - POST Create Event Missing Required Fields
   - POST Create Event Invalid Group ID
   - POST Create Event Invalid Date
   - DELETE Event
   - PUT Deleted Event Should Fail

## Expected vs Actual

**Expected:**  
- A valid event creation request succeeds with `201`.
- The created event appears in the event list.
- Updating the event succeeds with `200` and returns the updated fields.
- Invalid create requests are rejected with `400`.
- Deleting the event succeeds with `200`.
- Trying to update the deleted event fails with `404`.

**Actual:**  
1) **POST Create Event:** `201 Created`, returned the new event with an `_id`, title, group ID, date, location, and description ✅  
2) **GET Events:** `200 OK`, returned the events list successfully and included the created event ✅  
3) **PUT Update Event:** `200 OK`, returned the updated event with the new title, location, and description ✅  
4) **POST Create Event Missing Required Fields:** `400 Bad Request`, returned `"groupId, title, and startAt are required"` ✅  
5) **POST Create Event Invalid Group ID:** `400 Bad Request`, returned `"Invalid groupId"` ✅  
6) **POST Create Event Invalid Date:** `400 Bad Request`, returned `"Invalid startAt date"` ✅  
7) **DELETE Event:** `200 OK`, returned `"Event deleted"` ✅  
8) **PUT Deleted Event Should Fail:** `404 Not Found`, returned `"Event not found"` ✅  

## Outcome
**Pass/Fail:** Pass ✅
