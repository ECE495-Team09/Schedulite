## Why This Is an Integration Test

This test verifies that multiple parts of Schedulite work together in one real user flow. It uses the frontend UI, backend API routes, authentication/session behavior, event creation logic, RSVP logic, and MongoDB persistence. This is different from a unit test because it does not isolate one function. It is also different from the Postman route tests because it verifies that the actual user-facing app can trigger the backend behavior correctly.
