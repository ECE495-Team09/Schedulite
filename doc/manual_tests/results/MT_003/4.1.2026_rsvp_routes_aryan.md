# Manual Test Report

**Date (YYYY-MM-DD):** 2026-04-26  
**Tester:** Aryan Banerjee  

**Test Name:** Event RSVP routes — Create, Update, Invalid Input, and No-Auth in Postman  

**Test Steps (summary):**
1. Started backend + DB, ensured a test user existed, and obtained a valid JWT for that user.
2. Created or selected a test event in MongoDB and copied the event `_id`.
3. In Postman, set `Authorization: Bearer <JWT>` for the authenticated RSVP requests and removed auth for the no-auth request.
4. Ran these requests in order:
   - POST Create RSVP as `In`
   - PUT Update RSVP to `Maybe` with a note
   - POST RSVP with invalid status
   - POST RSVP with invalid event ID
   - POST RSVP with no auth
   - PUT RSVP for an event/user with no existing RSVP, if needed

## Expected vs Actual

**Expected:**  
- Authenticated RSVP create request succeeds and stores the RSVP.
- Authenticated RSVP update request succeeds and replaces the current user's previous RSVP.
- Invalid RSVP status is rejected with `400`.
- Invalid event ID is rejected with `400`.
- No-auth RSVP request is rejected with `401`.
- Updating an RSVP that does not exist returns `404`.

**Actual:**  
1) **POST Create RSVP as `In` (auth):** `201 Created`, returned `"message": "RSVP created"` and RSVP status `In` ✅  
2) **PUT Update RSVP to `Maybe` with note (auth):** `200 OK`, returned `"message": "RSVP updated"` and updated RSVP status/note ✅  
3) **POST RSVP With Invalid Status:** `400 Bad Request`, returned `"status must be one of: In, Out, Maybe"` ✅  
4) **POST RSVP With Invalid Event ID:** `400 Bad Request`, returned `"Invalid eventId"` ✅  
5) **POST RSVP With No Auth:** `401 Unauthorized`, returned `"Unauthorized"` ✅   

## Outcome
**Pass/Fail:** Pass ✅
